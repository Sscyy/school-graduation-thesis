## 第三章：基于非对称多尺度特征路由的细胞核实例分割

### 3.1 问题背景与动机

HoverNet 的解码器包含两个任务分支，它们的本质需求不同：

**NP Branch（核像素分割）**
- 任务性质：二分类（这个像素是核/不是核）
- 核心需求：**语义理解** ——需要知道"这一片区域整体是什么"
- 类比：看图说话，需要理解整体语义
- 更适合：**粗粒度（低分辨率）特征** ——语义更丰富，感受野更大

**HoVer Branch（水平/垂直距离图预测）**
- 任务性质：像素级回归（每个像素到其所在核质心的精确距离）
- 核心需求：**空间精度** ——需要精确知道"每个像素的位置"
- 类比：测量距离，需要精确的尺子
- 更适合：**细粒度（高分辨率）特征** ——空间细节更丰富

**现有方法（包括刘宣辰、原版HoverNet）的问题**：
所有方法都使用 ResNet50 这样的单尺度 encoder，两个分支只能用同一个尺度的特征，无法区分两种分支的不同需求。

**本方法的核心优势**：
SAM2.1 Hiera encoder 内置 FPN，天然输出三个尺度的特征。本章设计**非对称多尺度特征路由模块（Asymmetric Multi-scale Feature Routing，AMFR）**，让两个分支从三层特征中以不同权重汲取信息。

---

### 3.2 AMFR 模块设计

#### 3.2.1 特征路由的基本思路

SAM2.1 FPN 输出三层特征（输入 patch 为 256×256 时）：

| 特征层 | 空间分辨率 | 通道数 | 特点 |
|--------|-----------|--------|------|
| f₀     | 256×256   | 256    | 细粒度，空间细节最丰富，语义最弱 |
| f₁     | 128×128   | 256    | 中间层，平衡空间与语义 |
| f₂     | 64×64     | 256    | 粗粒度，语义最丰富，空间细节最弱 |

AMFR 模块的目标：对 NP Branch 和 HoVer Branch **分别学习一组权重**，决定三层特征各贡献多少。

#### 3.2.2 数学表达

对于 NP Branch，定义可学习权重向量：
$$\boldsymbol{\alpha}_{NP} = [\alpha_0, \alpha_1, \alpha_2], \quad \text{经 Softmax 归一化}$$

对于 HoVer Branch，定义独立的可学习权重向量：
$$\boldsymbol{\alpha}_{HV} = [\beta_0, \beta_1, \beta_2], \quad \text{经 Softmax 归一化}$$

融合后的 skip connection 特征（以同一空间分辨率为例）：
$$F_{NP} = \alpha_0 \cdot \text{Up}(f_0) + \alpha_1 \cdot \text{Up}(f_1) + \alpha_2 \cdot \text{Up}(f_2)$$
$$F_{HV} = \beta_0 \cdot \text{Up}(f_0) + \beta_1 \cdot \text{Up}(f_1) + \beta_2 \cdot \text{Up}(f_2)$$

其中 Up(·) 表示将特征上采样到目标分辨率。

**两个分支的权重完全独立**，互不干扰，由梯度反传自动学习。

#### 3.2.3 网络结构图

```
f₀ (256×256) ──────────────────────────────┐
                                            │
f₁ (128×128) ──Up(2×)──────────────────────┤
                                            │
f₂ (64×64)   ──Up(4×)──────────────────────┤
                                            │
            ┌──────────────────────────┐    │
            │    AMFR 模块              │    │
            │                          │    │
            │  NP 路由：Softmax([α₀,α₁,α₂])│◄──┘
            │  HV 路由：Softmax([β₀,β₁,β₂])│
            │                          │
            │  F_NP = α₀f₀+α₁f₁+α₂f₂  │
            │  F_HV = β₀f₀+β₁f₁+β₂f₂  │
            └────────┬──────────┬───────┘
                     │          │
              ┌──────▼──┐  ┌───▼──────┐
              │NP Branch│  │HoVer     │
              │Decoder  │  │Branch    │
              │         │  │Decoder   │
              └──────────┘  └──────────┘
```

#### 3.2.4 参数初始化

- 初始化时：`α₀ = α₁ = α₂ = 1/3`（均匀权重，等效于 baseline）
- 训练结束后，观察两组权重的分布验证假设：
  - 预期 NP Branch：`α₂ > α₁ > α₀`（语义权重偏大）
  - 预期 HoVer Branch：`β₀ > β₁ > β₂`（空间权重偏大）
  - 这个权重分布本身就是一个有趣的分析结果

#### 3.2.5 实现难度

**极低**。代码核心只有约 30 行：

```python
class AMFR(nn.Module):
    def __init__(self):
        super().__init__()
        # 两个分支各自的可学习权重（初始化为均匀）
        self.np_weights = nn.Parameter(torch.ones(3) / 3)
        self.hv_weights = nn.Parameter(torch.ones(3) / 3)
        
    def forward(self, f0, f1, f2, target_size):
        # 上采样到同一分辨率
        f1_up = F.interpolate(f1, size=target_size, mode='bilinear')
        f2_up = F.interpolate(f2, size=target_size, mode='bilinear')
        
        # Softmax 归一化权重
        np_w = F.softmax(self.np_weights, dim=0)
        hv_w = F.softmax(self.hv_weights, dim=0)
        
        # 加权融合
        F_np = np_w[0]*f0 + np_w[1]*f1_up + np_w[2]*f2_up
        F_hv = hv_w[0]*f0 + hv_w[1]*f1_up + hv_w[2]*f2_up
        
        return F_np, F_hv
```

### 3.3 第三章损失函数

沿用 HoverNet 原版损失（只保留 NP 和 HoVer 分支）：

$$\mathcal{L}_{total} = \mathcal{L}_{NP} + \mathcal{L}_{HV}$$

$$\mathcal{L}_{NP} = \mathcal{L}_{CE} + \mathcal{L}_{Dice}$$

$$\mathcal{L}_{HV} = \mathcal{L}_{MSE} + \mathcal{L}_{MSGE}$$

其中 MSGE 是梯度均方误差损失（HoverNet 原版设计，监督 HV 距离图的梯度）。

## 第四章：基于边缘感知跳跃连接的高泛化性细胞核分割

### 4.1 问题背景与动机

第三章方法在单数据集上取得了较好的精度，但在**跨数据集**场景下性能会下降。主要原因：

1. **细胞核边界模糊**：不同染色方案、不同组织类型下，核与背景的边界清晰度差异大
2. **颜色/纹理差异大**：不同医院、不同染色批次的切片颜色差异显著，模型容易过拟合颜色信息
3. **边界信息利用不足**：HoverNet 的 skip connection 直接传递来自编码器的特征，没有专门强化边界信息

**解决思路**：

- 引入**边缘引导分支（Edge Branch）**：用额外的解码头专门预测核边缘，强迫模型学习边界几何形态，而非颜色纹理
- 设计**边缘感知跳跃连接（Edge-Aware Skip Connection，EA-Skip）**：在 skip connection 里对细粒度特征做主动边缘增强，让解码器直接接收到更丰富的边界信息

**为什么边缘特征对泛化有帮助？**

核的几何形状（圆形/椭圆形）在不同染色条件下是**不变的**，但颜色/纹理会随染色方案大幅变化。边缘信息反映几何形态而非颜色，因此天然具有跨域稳定性。

---

### 4.2 边缘引导分支（Edge Branch）

#### 4.2.1 边缘 GT 的生成

对每张训练图的核分割掩码，用**拉普拉斯算子**自动提取边缘：

```python
import cv2
import numpy as np

def get_edge_gt(mask):
    """
    mask: 二值核分割掩码 (H, W)，核=1，背景=0
    返回: 二值边缘图 (H, W)，边缘=1，其余=0
    """
    mask_uint8 = (mask * 255).astype(np.uint8)
    laplacian = cv2.Laplacian(mask_uint8, cv2.CV_64F)
    edge = (np.abs(laplacian) > 0).astype(np.float32)
    return edge
```

这个方法**无需人工标注**，直接从现有掩码标注派生，不增加任何额外的数据标注成本。

#### 4.2.2 Edge Branch 网络结构

Edge Branch 是一个轻量的额外解码头，与 NP Branch 共享编码器特征，独立解码：

```
编码器输出特征 (经过 AMFR 路由)
        ↓
  Edge Branch Decoder
  (结构与 NP Branch 相同，但权重独立)
        ↓
  1×1 卷积 → Sigmoid
        ↓
  边缘预测图 (B, 1, H, W)，值域 [0, 1]
```

Edge Branch 只在**训练时**使用，推理时不输出边缘图（不改变推理速度和输出格式）。

#### 4.2.3 Edge Branch 损失函数

边缘图存在严重的类别不平衡（边缘像素远少于背景像素），因此使用 **Dice Loss** 而非交叉熵：

$$\mathcal{L}_{Edge} = \mathcal{L}_{Dice}(\hat{E}, E_{gt})$$

$$\mathcal{L}_{Dice} = 1 - \frac{2 \sum \hat{E} \cdot E_{gt}}{\sum \hat{E} + \sum E_{gt} + \epsilon}$$

### 4.3 边缘感知跳跃连接（EA-Skip）

这是第四章**区别于刘宣辰**的关键创新点。

#### 4.3.1 普通 Skip Connection 的局限

标准 skip connection 直接把编码器特征传给解码器：

```
f₀ (细粒度, 256×256) ──────直接→ NP Branch Decoder
```

这些特征包含了图像中的所有信息（颜色、纹理、结构），解码器需要从中自己判断哪些是有用的。在跨域场景下，颜色/纹理信息会产生干扰。

#### 4.3.2 EA-Skip 的设计

EA-Skip 在 **Level 0 细粒度特征**（256×256，最敏感的空间细节层）进入 skip connection 之前，先经过一个**边缘感知增强卷积**：

```
f₀ (256×256, 256ch)
        ↓
  边缘感知增强模块 (EAM)
  ├── Branch A: 3×3 可学习卷积 (初始化为 Laplacian 卷积核)
  │                         ↓ 高频/边缘成分
  └── Branch B: 1×1 卷积保留原始特征
                            ↓ 低频/语义成分
        ↓ 两路相加融合
  f₀_enhanced (256×256, 256ch)
        ↓
  送入 NP Branch 和 HoVer Branch 的 Skip Connection
```

**Branch A 的初始化**：将 3×3 卷积核的权重初始化为拉普拉斯算子形式：

```python
laplacian_init = torch.tensor([
    [0,  1, 0],
    [1, -4, 1],
    [0,  1, 0]
], dtype=torch.float32)

# 初始化所有输出通道的所有输入通道卷积核为 Laplacian
conv.weight.data = laplacian_init.unsqueeze(0).unsqueeze(0).repeat(256, 256, 1, 1)
```

虽然初始化为 Laplacian，但这是**可学习**的，训练中会自动调整为对分割任务最优的边缘检测形式。

#### 4.3.3 为什么只在 Level 0 做边缘增强？

- Level 0（256×256）：分辨率最高，包含最丰富的边界细节，边缘增强效果最显著
- Level 1（128×128）、Level 2（64×64）：分辨率较低，边缘信息已被部分压缩，增强效果有限且增加计算量

只在一个层上做增强，**实现简单，效果集中，消融实验清晰**。

#### 4.3.4 EAM 模块完整代码

```python
class EdgeAwareModule(nn.Module):
    """边缘感知增强模块 (EAM)"""
    def __init__(self, channels=256):
        super().__init__()
        # Branch A: 可学习边缘检测卷积
        self.edge_conv = nn.Conv2d(channels, channels, 3, padding=1, groups=channels)
        # Branch B: 语义保留
        self.semantic_conv = nn.Conv2d(channels, channels, 1)
        # 融合后的归一化
        self.bn = nn.BatchNorm2d(channels)
        self.relu = nn.ReLU(inplace=True)
        
        # 初始化 Branch A 为 Laplacian
        self._init_laplacian()
        
    def _init_laplacian(self):
        lap = torch.tensor([[0, 1, 0],
                             [1,-4, 1],
                             [0, 1, 0]], dtype=torch.float32)
        self.edge_conv.weight.data = lap.view(1, 1, 3, 3).repeat(256, 1, 1, 1)
        
    def forward(self, x):
        edge_feat = self.edge_conv(x)       # 高频边缘成分
        semantic_feat = self.semantic_conv(x)  # 低频语义成分
        out = self.relu(self.bn(edge_feat + semantic_feat))
        return out
```

### 4.4 第四章完整损失函数

在第三章损失的基础上，加入边缘分支损失：

$$\mathcal{L}_{total} = \mathcal{L}_{NP} + \mathcal{L}_{HV} + \lambda \cdot \mathcal{L}_{Edge}$$

其中 $\lambda$ 是边缘损失权重，初始设为 1，可通过消融确定最优值（通常取 0.5～2.0 之间）。

**可选改进**：参考刘宣辰的动态损失平衡思路，将 $\lambda$ 设为可学习参数：

$$\mathcal{L}_{total} = \frac{1}{2\sigma_1^2}\mathcal{L}_{NP} + \frac{1}{2\sigma_2^2}\mathcal{L}_{HV} + \frac{1}{2\sigma_3^2}\mathcal{L}_{Edge} + \log\sigma_1\sigma_2\sigma_3$$

其中 $\sigma_1, \sigma_2, \sigma_3$ 是可学习标量。这个公式直接从刘宣辰论文的推导里来，可以直接引用。

## 你的核心差异化来源（与所有对比论文的区别）

1. **vs 刘宣辰**：她用 ResNet50 单尺度，没有条件做多尺度路由；你的 AMFR 完全建立在 SAM2.1 FPN 三尺度这一独有前提上
2. **vs CellViT**：CellViT 用 SAM1 encoder + U-Net decoder，没有 HoverNet 的 HoVer map 机制；你保留了 HoVer map，并专门设计了针对 HoVer Branch 需要更多细粒度特征这一特点的路由
3. **vs PanopMamba/CPMamba**：他们用 Mamba，你不用，反而设计了更简洁且动机更清晰的多尺度路由
